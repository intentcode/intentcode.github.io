The Solution

Intent: Documentation that Lives with Code

Intent files are structured explanations stored alongside your code. They describe the 'why' behind changes using semantic anchors that survive refactoring.

  • ‚úì Decisions: Document the rationale behind choices

How It Works

Simple setup, powerful results

1

Create Intent Files

Add a .intent/ folder to your repo. Write markdown files describing your changes with semantic anchors.

2

Link to Code

Use @function:name, @class:Name, or @pattern:text to anchor explanations to code locations that survive refactoring.

3

Review with Context

Open Intent to see diffs alongside their explanations. Intent cards appear right next to the code they describe.
